## 문제
머신러닝에서 '분류(Classification)'와 '회귀(Regression)'의 차이에 대한 설명으로 올바른 것은?
1. 분류는 연속적인 수치를 예측하고, 회귀는 범주를 예측한다
2. 분류는 범주(스팸/정상 등)를 예측하고, 회귀는 연속적인 수치(가격 등)를 예측한다
3. 분류와 회귀는 동일한 개념으로 혼용해서 사용한다
4. 분류는 비지도학습이고, 회귀는 지도학습이다
5. 분류는 텍스트에만 적용되고, 회귀는 숫자에만 적용된다

## 정답
2

## 해설
1. **오답** - 분류와 회귀의 역할이 반대로 설명되어 있습니다.
2. **정답** - 분류(Classification)는 이메일의 스팸 여부, 이미지 속 동물 종류처럼 범주를 예측합니다. 회귀(Regression)는 주택 가격, 기온처럼 연속적인 수치를 예측합니다.
3. **오답** - 분류와 회귀는 목적과 출력 형태가 다른 별개의 개념입니다.
4. **오답** - 분류와 회귀 모두 지도학습(Supervised Learning)의 대표적인 유형입니다.
5. **오답** - 분류와 회귀 모두 텍스트, 이미지, 숫자 등 다양한 데이터에 적용 가능합니다.

## 문제
Word2Vec에 대한 설명으로 **틀린** 것은?
1. '비슷한 맥락의 단어는 비슷한 의미를 가진다'는 가정에 기반한다
2. King - Man + Woman = Queen과 같은 벡터 연산이 가능하다
3. 모든 문맥에서 동일한 단어는 동일한 벡터를 가진다
4. CBOW와 Skip-gram 두 가지 학습 방식이 있다
5. 각 문맥에 따라 단어의 벡터가 다르게 생성된다

## 정답
5

## 해설
1. **옳음** - Word2Vec은 분포 가설(Distributional Hypothesis)에 기반합니다.
2. **옳음** - Word2Vec의 대표적인 특성으로, 단어 간 의미 관계가 벡터 연산으로 표현됩니다.
3. **옳음** - Word2Vec의 한계점입니다. 동일한 단어는 문맥과 관계없이 같은 벡터를 가집니다.
4. **옳음** - CBOW(주변 → 중심 예측)와 Skip-gram(중심 → 주변 예측) 방식이 있습니다.
5. **틀림(정답)** - Word2Vec은 문맥에 따라 벡터가 달라지지 않습니다. 문맥별 다른 벡터를 제공하는 것은 BERT, ELMo 같은 동적 임베딩 모델입니다.

## 문제
토큰화(Tokenization)에서 한국어가 영어보다 토큰 효율이 낮은 주된 이유는?
1. 한국어 문장이 영어보다 항상 길기 때문에
2. 한국어는 교착어로 조사가 붙어 어휘 수가 폭발적으로 증가하기 때문에
3. 한국어에는 특수문자가 많기 때문에
4. 토크나이저가 한국어를 지원하지 않기 때문에
5. 한국어는 띄어쓰기가 없기 때문에

## 정답
2

## 해설
1. **오답** - 문장 길이 자체가 토큰 효율의 직접적인 원인은 아닙니다.
2. **정답** - 한국어는 교착어로 '가다'에 조사가 붙으면 '가다, 가서, 가니, 가면, 갔다, 갔니, 가더라' 등 수많은 변형이 생깁니다. 영어는 'go, goes, went, going' 정도이지만 한국어는 변형이 훨씬 많아 토큰 수가 증가합니다.
3. **오답** - 특수문자는 토큰 효율에 큰 영향을 주지 않습니다.
4. **오답** - 대부분의 현대 토크나이저는 한국어를 지원합니다.
5. **오답** - 한국어도 띄어쓰기가 있습니다.

## 문제
Transformer 아키텍처에서 Self-Attention의 역할로 가장 적절한 것은?
1. 입력 데이터를 토큰으로 분할하는 역할
2. 문장 내 각 단어가 다른 단어들과 어떤 관계를 가지는지 파악하는 역할
3. 모델의 파라미터를 초기화하는 역할
4. 출력 토큰을 하나씩 순차적으로 생성하는 역할
5. 학습 데이터를 전처리하는 역할

## 정답
2

## 해설
1. **오답** - 토큰 분할은 토크나이저의 역할입니다.
2. **정답** - Self-Attention은 문장 내 각 단어(토큰)가 다른 모든 단어와 어떤 관계를 가지는지 동시에 파악합니다. 예를 들어 "The cat sat on the mat"에서 'sat'이 'cat'과 어떤 관련이 있는지 파악합니다.
3. **오답** - 파라미터 초기화는 학습 시작 전 과정입니다.
4. **오답** - 순차적 토큰 생성은 Auto-Regressive 방식의 특성입니다.
5. **오답** - 데이터 전처리는 모델 외부에서 이루어집니다.

## 문제
Transformer에서 Positional Encoding이 필요한 이유는?
1. 모델의 학습 속도를 높이기 위해
2. Self-Attention이 병렬 처리되므로 토큰의 순서 정보를 별도로 제공하기 위해
3. 모델의 메모리 사용량을 줄이기 위해
4. 다국어 처리를 가능하게 하기 위해
5. 출력 토큰의 품질을 높이기 위해

## 정답
2

## 해설
1. **오답** - Positional Encoding의 주목적은 학습 속도 향상이 아닙니다.
2. **정답** - RNN은 순차 처리로 자연스럽게 순서 정보를 가지지만, Transformer의 Self-Attention은 모든 토큰을 병렬로 처리하므로 순서 정보가 사라집니다. Positional Encoding은 각 토큰의 위치(순서) 정보를 임베딩에 더해줍니다.
3. **오답** - 메모리 사용량과는 관련이 없습니다.
4. **오답** - 다국어 처리와는 직접적인 관련이 없습니다.
5. **오답** - 출력 품질 향상이 아닌 순서 정보 제공이 목적입니다.

## 문제
Multi-Head Attention에 대한 설명으로 올바른 것은?
1. 여러 개의 Attention을 순차적으로 실행하여 성능을 높인다
2. 여러 Head가 각기 다른 관점에서 동시에 단어 관계를 분석한다
3. Head 수가 많을수록 항상 성능이 좋아진다
4. Multi-Head는 Encoder에서만 사용된다
5. 각 Head의 출력은 평균을 내어 최종 결과를 만든다

## 정답
2

## 해설
1. **오답** - Multi-Head Attention은 순차적이 아닌 병렬로 실행됩니다.
2. **정답** - 여러 Head가 각각 다른 관점(문법, 의미, 지시어 관계 등)에서 동시에 단어 간 관계를 분석합니다. 이를 통해 다양한 특징을 포착할 수 있습니다.
3. **오답** - Head 수가 많다고 항상 좋은 것은 아니며, 적절한 수를 선택해야 합니다.
4. **오답** - Encoder와 Decoder 모두에서 사용됩니다.
5. **오답** - 각 Head의 출력은 평균이 아닌 연결(Concatenation) 후 선형 변환됩니다.

## 문제
Transformer의 FFNN(Feed Forward Neural Network)의 역할로 가장 적절한 것은?
1. 토큰 간의 관계를 파악하는 역할
2. Self-Attention이 놓친 비선형적인 복잡한 관계를 학습하는 역할
3. 입력 데이터를 임베딩하는 역할
4. 출력 토큰의 확률을 계산하는 역할
5. 학습률을 조절하는 역할

## 정답
2

## 해설
1. **오답** - 토큰 간 관계 파악은 Self-Attention의 역할입니다.
2. **정답** - FFNN은 비선형 활성화 함수를 통해 Self-Attention이 놓친 복잡한 비선형 관계를 학습합니다. Self-Attention은 선형 연산 위주이므로 FFNN이 이를 보완합니다.
3. **오답** - 임베딩은 입력 계층에서 이루어집니다.
4. **오답** - 확률 계산은 최종 출력 계층(Softmax)에서 이루어집니다.
5. **오답** - 학습률 조절은 옵티마이저의 역할입니다.

## 문제
BERT와 GPT의 구조적 차이에 대한 설명으로 올바른 것은?
1. BERT는 Decoder 기반, GPT는 Encoder 기반이다
2. BERT는 단방향, GPT는 양방향으로 문맥을 파악한다
3. BERT는 Encoder 기반으로 양방향 문맥을 파악하고, GPT는 Decoder 기반으로 단방향 생성에 특화되어 있다
4. BERT와 GPT는 동일한 구조로 학습 데이터만 다르다
5. BERT는 텍스트 생성, GPT는 문장 분류에 주로 사용된다

## 정답
3

## 해설
1. **오답** - BERT는 Encoder 기반, GPT는 Decoder 기반입니다. 역으로 설명되어 있습니다.
2. **오답** - BERT는 양방향, GPT는 단방향입니다. 역으로 설명되어 있습니다.
3. **정답** - BERT는 Transformer의 Encoder만 사용하여 양방향으로 문맥을 파악하므로 문장 이해, 분류에 적합합니다. GPT는 Decoder만 사용하여 왼쪽에서 오른쪽으로 단방향 생성에 특화되어 있습니다.
4. **오답** - 구조 자체가 다릅니다.
5. **오답** - 역할이 반대입니다. BERT는 분류, GPT는 생성에 주로 사용됩니다.

3. In-Context Learning과 Few-Shot Learning

3

2. SFT(Supervised Fine-Tuning)와 RLHF(강화학습)를 적용하여 질의응답 능력이 크게 향상되었다

## 문제
SFT(Supervised Fine-Tuning)와 RLHF(Reinforcement Learning from Human Feedback)에 대한 설명으로 올바른 것은?
1. SFT는 강화학습 기반, RLHF는 지도학습 기반이다
2. SFT는 고품질 데이터로 행동을 가르치고, RLHF는 인간 선호도 기반으로 미세 조정한다
3. SFT와 RLHF는 동시에 적용할 수 없다
4. RLHF가 SFT보다 먼저 수행된다
5. SFT는 Pre-training 단계에서 수행된다

## 정답
2

## 해설
1. **오답** - SFT는 지도학습 기반, RLHF는 강화학습 기반입니다.
2. **정답** - SFT는 고품질의 질문-답변 쌍으로 모델이 어떻게 응답해야 하는지 가르칩니다. RLHF는 인간이 여러 응답 중 선호하는 것을 선택하면 이를 바탕으로 보상 모델을 학습하고 PPO 등으로 미세 조정합니다.
3. **오답** - 일반적으로 SFT 후 RLHF를 순차적으로 적용합니다.
4. **오답** - SFT가 RLHF보다 먼저 수행됩니다 (Pre-training → SFT → RLHF 순서).
5. **오답** - SFT는 Pre-training 이후의 Fine-tuning 단계에서 수행됩니다.

## 문제
MoE(Mixture of Experts) 구조에 대한 설명으로 **틀린** 것은?
1. 여러 전문가 네트워크 중 일부만 활성화하여 계산 효율을 높인다
2. Router가 입력에 따라 어떤 전문가를 활성화할지 결정한다
3. 전체 파라미터 중 일부만 사용하므로 추론 비용이 절감된다
4. 모든 전문가를 항상 동시에 활성화하여 성능을 극대화한다
5. GPT-4에서 사용된 것으로 알려져 있다

## 정답
4

## 해설

2. **옳음** - Router(또는 Gating Network)가 입력 토큰에 따라 적절한 전문가를 선택합니다.

5. **옳음** - GPT-4에서 MoE 구조가 사용된 것으로 알려져 있습니다.

## 문제
GPT-o1, o3 같은 OpenAI의 o-시리즈 모델의 특징으로 가장 적절한 것은?
1. 가장 빠른 추론 속도를 목표로 설계되었다
2. 추론(Reasoning) 특화 모델로, 내부적으로 긴 사고 과정을 거친다
3. 멀티모달 처리를 완전히 배제한 텍스트 전용 모델이다
4. 파라미터 수를 최소화한 경량 모델이다
5. 코드 생성 기능을 제거한 모델이다

## 정답
2

## 해설
1. **오답** - o-시리즈는 속도보다 추론 능력에 중점을 둡니다.

2. 7B 모델은 약 14GB의 VRAM이 필요하다
3. 7B 모델은 약 28GB의 VRAM이 필요하다
4. 7B 모델은 약 70GB의 VRAM이 필요하다

2. 프롬프트에 소수의 예시를 포함하여 모델이 패턴을 파악하고 수행하게 하는 방식이다
3. 예시 없이 태스크를 수행하는 Zero-Shot과 동일한 개념이다
4. 모델의 파라미터를 조정해야만 가능한 방식이다
5. 오직 최신 모델에서만 지원되는 기능이다

## 정답
2

## 해설
1. **오답** - Few-Shot은 소수(few)의 예시만 사용합니다.
2. **정답** - Few-Shot Learning은 프롬프트에 2~5개 정도의 예시를 포함하여, 모델이 해당 패턴을 파악하고 새로운 입력에 대해 유사하게 응답하도록 하는 방식입니다. 파라미터 업데이트 없이 작동합니다.
3. **오답** - Zero-Shot은 예시가 없고, Few-Shot은 소수의 예시가 있습니다.
4. **오답** - 파라미터 조정 없이 프롬프트만으로 작동합니다.
5. **오답** - GPT-3부터 가능한 기능입니다.
