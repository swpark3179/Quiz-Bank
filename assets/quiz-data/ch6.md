## 문제
Fine Tuning에 대한 설명으로 가장 적절한 것은?
1. 모델을 처음부터 대규모 데이터로 학습시키는 것이다
2. 이미 학습된 LLM에 새로운 데이터를 추가로 학습시켜 특정 목적에 맞게 성능을 향상시키는 것이다
3. 프롬프트에 Context를 추가하는 것이다
4. 모델의 추론 속도를 높이는 것이다
5. 모델의 파라미터 수를 줄이는 것이다

## 정답
2

## 해설
1. **오답** - 처음부터 학습시키는 것은 Pre-training입니다.
2. **정답** - Fine Tuning은 이미 학습된 대규모 언어 모델(LLM)에 새로운 데이터를 추가로 학습시켜 특정 목적에 맞게 성능을 향상시키는 방법입니다. Pre-training이 초중고 일반 교육이라면, Fine Tuning은 대학원에서 특정 전문 분야를 심화 학습하는 것과 같습니다.
3. **오답** - Context 추가는 Prompt Engineering입니다.
4. **오답** - 추론 속도 향상은 양자화 등의 최적화 기법입니다.
5. **오답** - 파라미터 수 감소는 Pruning이나 양자화입니다.

## 문제
Prompt Engineering과 Fine Tuning의 차이에 대한 설명으로 올바른 것은?
1. 둘 다 모델의 파라미터를 변경한다
2. Prompt Engineering은 모델 자체의 파라미터를 조정하고, Fine Tuning은 입력 텍스트를 변경한다
3. Prompt Engineering은 입력 텍스트를 변경하고, Fine Tuning은 모델 자체의 파라미터를 조정한다
4. 둘 다 매번 Context 제공이 필요하다
5. 둘 다 1회성 비용이 발생한다

## 정답
3

## 해설
1. **오답** - Prompt Engineering은 파라미터를 변경하지 않습니다.
2. **오답** - 역할이 반대로 설명되어 있습니다.
3. **정답** - Prompt Engineering은 입력 텍스트(프롬프트) 안에서 Context를 추가하는 방식이고, Fine Tuning은 모델 자체의 파라미터를 조정하여 "사고 회로"를 변경합니다. 비유하면 Prompt Engineering은 "매번 설명서를 주며 일시키는 것"이고, Fine Tuning은 "아예 그 지식을 머릿속에 심어주는 것"입니다.
4. **오답** - Fine Tuning은 한 번 학습하면 영구 적용됩니다.
5. **오답** - Prompt Engineering은 런타임 비용(토큰당 과금)이 발생합니다.

## 문제
Pre-training에 대한 설명으로 올바른 것은?
1. 소량의 데이터로 빠르게 학습하는 과정이다
2. 인터넷의 방대한 텍스트 데이터를 "다음 단어 예측" 방식으로 학습하는 과정이다
3. 질문-답변 쌍으로 학습하는 과정이다
4. 인간 피드백을 활용하는 과정이다
5. 모델을 압축하는 과정이다

## 정답
2

## 해설
1. **오답** - Pre-training은 초대량의 데이터가 필요합니다 (수조 개의 토큰).
2. **정답** - Pre-training은 LLM을 처음부터 학습시키는 과정으로, 인터넷의 방대한 텍스트 데이터(위키피디아, 뉴스, 책, 코드 등)를 "다음 단어 예측"(Next Token Prediction) 방식으로 학습합니다. 수천 대의 GPU와 수개월의 시간, 수백억~수천억 원의 비용이 소요됩니다.
3. **오답** - 질문-답변 쌍 학습은 Instruction Tuning입니다.
4. **오답** - 인간 피드백 활용은 RLHF입니다.
5. **오답** - 모델 압축은 양자화, Pruning 등입니다.

## 문제
Fine Tuning이 Pre-training 대비 효율적인 이유로 올바른 것은?
1. Fine Tuning이 더 많은 데이터가 필요하기 때문에
2. Base Model이 이미 언어의 기본 구조, 문법, 일반 상식을 알고 있어 특정 능력만 추가하면 되기 때문에
3. Fine Tuning이 더 많은 GPU가 필요하기 때문에
4. Fine Tuning이 더 오랜 시간이 걸리기 때문에
5. Fine Tuning은 학습이 불가능하기 때문에

## 정답
2

## 해설
1. **오답** - Fine Tuning은 훨씬 적은 데이터가 필요합니다 (수천~수십만 개).
2. **정답** - Fine Tuning이 효율적인 이유는 Base Model이 이미 언어의 기본 구조, 문법, 일반 상식을 알고 있기 때문입니다. 이 기반 위에 특정 능력만 추가하면 되므로 빠른 수렴(몇 시간~며칠)과 낮은 비용(수십만~수천만 원)으로 가능합니다.
3. **오답** - Fine Tuning은 적은 GPU로 가능합니다.
4. **오답** - Fine Tuning은 시간이 짧게 걸립니다.
5. **오답** - Fine Tuning은 학습이 가능합니다.

3

4. **오답** - 이것은 Thinking Model의 설명입니다.

## 문제
Base Model의 한계에 대한 설명으로 올바른 것은?
1. 질문에 정확하게 답변한다
2. "다음 단어 예측"만 학습했기 때문에 질문에 답하는 것이 아니라 문장을 이어가는 방식으로 응답한다
3. 대화 능력이 뛰어나다
4. 안전한 응답만 생성한다
5. 모든 언어를 완벽하게 지원한다

## 정답
2

## 해설
1. **오답** - Base Model은 질문에 정확히 답하지 못합니다.
2. **정답** - Base Model은 "다음 단어 예측"만 학습했기 때문에, 질문을 하면 질문에 "답하는 것"이 아니라 인터넷에서 봤던 패턴대로 "문장을 이어가는 것"입니다. 예를 들어 "파리의 수도는 어디야?"라고 물으면 이상하게 답할 수 있습니다.
3. **오답** - 대화 능력은 Instruction Tuning 후에 생깁니다.
4. **오답** - 안전성은 Alignment 후에 강화됩니다.
5. **오답** - 언어 능력은 학습 데이터에 따라 다릅니다.

## 문제
Fine Tuning의 4가지 핵심 목적이 **아닌** 것은?
1. 지식 (Knowledge) - 최신 정보나 도메인 지식 주입
2. 능력 (Capability) - 특정 작업 수행 능력 향상
3. 형식 (Format) - 출력 스타일, 길이, 말투 조정
4. 안전 (Safety) - 유해한 요청 거부, 윤리적 응답
5. 속도 (Speed) - 추론 속도 향상

## 정답
5

## 해설
1. **목적에 해당** - 모델이 모르는 최신 정보나 도메인 지식을 가르칩니다.
2. **목적에 해당** - 수학, 코딩, 요약, 번역 등 특정 작업 수행 능력을 향상시킵니다.
3. **목적에 해당** - JSON 형식 응답, 말투 조정, 길이 제한 등 출력 스타일을 조정합니다.
4. **목적에 해당** - Alignment를 통해 모델을 인간의 가치관에 맞게 조정합니다.
5. **정답(목적 아님)** - 추론 속도 향상은 Fine Tuning의 목적이 아닙니다. 속도는 양자화 등의 최적화 기법으로 개선합니다.

SFT(Supervised Fine-Tuning)에 대한 설명으로 올바른 것은?
1. 강화 학습을 사용하는 방법이다
2. 정답이 있는 데이터를 사용해 모델을 학습시키는 방식이다
3. 인간 피드백을 활용하는 방법이다
4. 보상 모델을 학습시키는 방법이다
5. 모델을 처음부터 학습시키는 방법이다

2

## 해설

SFT의 한계로 올바른 것은?
1. 학습이 너무 빠르다
2. 정답 데이터가 필요하고, 모든 답을 "100% 정답"으로 취급하여 미묘한 차이 표현이 어렵다
3. 구현이 너무 복잡하다
4. 예측이 불가능하다
5. 강화 학습보다 비용이 높다

2

4. **오답** - 예측 가능한 결과가 장점입니다.
5. **오답** - SFT가 RL보다 비용이 낮습니다.

## 문제
CPT(Continuous Pre-Training)에 대한 설명으로 올바른 것은?
1. 질문-답변 쌍으로 학습하는 방식이다
2. 이미 Pre-trained된 모델에 새로운 대량의 텍스트를 추가로 학습시키는 방식이다
3. 소량의 데이터로 학습하는 방식이다
4. 인간 피드백을 활용하는 방식이다
5. 모델을 압축하는 방식이다

## 정답
2

## 해설
1. **오답** - 질문-답변 쌍은 SFT입니다.
2. **정답** - CPT(Continuous Pre-Training)는 이미 Pre-trained된 모델에 새로운 대량의 텍스트를 추가로 학습시키는 방식입니다. Pre-training과 동일한 "다음 단어 예측" 방식을 사용하며, 언어/지식의 기반 자체를 변화시킵니다. 비유하면 새로운 언어를 배우는 것과 같습니다.
3. **오답** - CPT는 대량의 데이터가 필요합니다 (수십억 토큰).
4. **오답** - 인간 피드백은 RLHF입니다.
5. **오답** - 압축은 양자화입니다.

CPT 활용 사례로 올바른 것은?
1. ChatGPT의 대화 능력 강화

2. 기존 가중치 W는 고정하고, 저차원 행렬 B×A만 학습하여 W + B×A로 출력한다

2. **정답** - RL vs Distillation: RL은 새로운 능력 개발이 가능하지만 비용이 높고 불안정합니다. Distillation은 빠르고 안정적이지만 선생님 능력에 제한됩니다. 작은 모델을 처음부터 RL로 학습시키면 Distillation보다 훨씬 비용이 들고 성능도 낮습니다.
3. **오답** - 두 방법은 다릅니다.
4. **오답** - Distillation도 좋은 성능을 보입니다.
5. **오답** - Distillation이 더 안정적입니다.

## 문제
SFT와 RL의 최종 결론으로 올바른 것은?
1. SFT만으로 충분하다
2. RL만으로 충분하다
3. SFT + RL 결합이 최선이며, 각각의 장점(SFT: 지식/형식 학습, RL: 추론 경로 최적화)을 살린 시너지 효과가 필요하다
4. 둘 다 필요 없다
5. Pre-training만 필요하다

## 정답
3

## 해설
1. **오답** - SFT만으로는 매끄러운 추론까지 어렵습니다.
2. **오답** - RL만으로는 외부 지식 없이 근본적 한계가 있습니다.
3. **정답** - 최종 결론: 1. SFT만으로는 부족 - 지식은 배우지만 매끄러운 추론까지는 어려움, 2. RL만으로도 부족 - 외부 지식 없이는 근본적 한계 존재, 3. SFT + RL 결합이 최선 - SFT(지식/추론 패턴/형식 습득) + RL(추론 경로 최적화/지식 정확성 향상/일반화 능력 강화)의 시너지 효과, 4. 베이스 모델이 중요 - 아무리 좋은 Fine Tuning도 기반 모델의 한계를 넘기 어려움
4. **오답** - Fine Tuning이 필요합니다.
5. **오답** - Fine Tuning도 필요합니다.

## 문제
RL이 SFT와 다르게 기여하는 점으로 올바른 것은?
1. RL은 형식만 개선한다
2. RL은 올바른 지식을 꺼내는 경로를 선택하도록 유도하여 추론 경로를 최적화한다
3. RL은 학습이 불가능하다
4. RL은 SFT와 동일하다
5. RL은 메모리만 절약한다

## 정답
2

## 해설
1. **오답** - 형식 개선은 SFT로도 가능합니다.
2. **정답** - 실험 결과, RL은 단순히 "형식"만 개선하는 것이 아니라 올바른 지식을 꺼내는 경로를 선택하도록 유도합니다. SFT만 적용한 모델이 오답을 낸 경우에도 SFT+RL 모델은 정답을 맞추는 경우가 있는데, 이는 RL이 추론 경로를 최적화하기 때문입니다.
3. **오답** - RL 학습이 가능합니다.
4. **오답** - RL과 SFT는 다릅니다.
5. **오답** - 메모리 절약이 주목적이 아닙니다.
