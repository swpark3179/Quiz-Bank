## 문제 1
### **LLM의 Knowledge Cutoff(지식 마감일)에 대한 설명으로 올바른 것은?**

| 보기 | 설명 |
|------|------|
| 1. LLM이 새로운 정보를 학습할 수 있는 최대 용량이다 | |
| **2. LLM이 학습 데이터의 마지막 시점까지만 알고 있어 그 이후 정보는 답할 수 없다** | |
| 3. LLM의 응답 시간 제한이다 | |
| 4. LLM이 처리할 수 있는 최대 토큰 수이다 | |
| 5. LLM의 파라미터 업데이트 주기이다 | |

**정답: 2**

**해설:**
1. **오답** - 학습 용량이 아닌 학습 시점의 한계입니다.
2. **정답** - Knowledge Cutoff는 LLM이 학습 데이터의 마지막 시점까지만 알고 있다는 의미입니다. 예를 들어 GPT-4는 2024년 6월, Claude는 2025년 1월까지의 정보만 알고 있어 "어제 있었던 축구 경기 결과"나 "오늘 발표된 신제품 정보"에 대해서는 답할 수 없습니다.
3. **오답** - 응답 시간과는 무관합니다.
4. **오답** - 최대 토큰 수는 Context Length입니다.
5. **오답** - 파라미터 업데이트 주기와는 다릅니다.

## 문제 2
### **Hallucination(환각) 현상에 대한 설명으로 올바른 것은?**

| 보기 | 설명 |
|------|------|
| 1. LLM이 응답을 거부하는 현상이다 | |
| **2. LLM이 실제로 존재하지 않는 정보를 그럴듯하게 만들어내는 현상이다** | |
| 3. LLM이 너무 느리게 응답하는 현상이다 | |
| 4. LLM이 같은 답변을 반복하는 현상이다 | |
| 5. LLM이 언어를 인식하지 못하는 현상이다 | |

**정답: 2**

**해설:**
1. **오답** - 응답 거부는 Hallucination이 아닙니다.
2. **정답** - Hallucination(환각)은 LLM이 마치 꿈을 꾸듯이 실제로 존재하지 않는 정보를 그럴듯하게 만들어내는 현상입니다. 답변이 매우 자연스럽고 자신감 있게 들리지만 내용이 틀릴 수 있어, 사용자가 진짜인지 가짜인지 구분하기 어렵습니다.
3. **오답** - 응답 속도와는 무관합니다.
4. **오답** - 반복은 다른 현상입니다.
5. **오답** - 언어 인식 문제와는 다릅니다.

|------|------|

| **2. 검색 + In-Context Learning** | |
| 3. 모델의 파라미터를 변경하는 것 | |
| 4. 여러 모델을 앙상블하는 것 | |

3. **오답** - 파라미터 변경은 파인튜닝입니다.

## 문제 5
### **In-Context Learning에 대한 설명으로 올바른 것은?**

| 보기 | 설명 |
|------|------|
| 1. LLM의 파라미터가 업데이트되는 학습 방식이다 | |
| **2. LLM이 프롬프트에 포함된 정보를 이해하고 활용하여 답변을 생성하는 것으로, 파라미터는 변하지 않는다** | |
| 3. 대규모 데이터로 사전학습하는 방식이다 | |
| 4. GPU를 사용한 병렬 학습 방식이다 | |
| 5. 오프라인에서만 가능한 학습 방식이다 | |

**정답: 2**

**해설:**
1. **오답** - In-Context Learning은 파라미터를 업데이트하지 않습니다.
2. **정답** - In-Context Learning은 LLM이 프롬프트에 포함된 정보(Context)를 이해하고 활용하여 답변을 생성하는 능력입니다. 중요한 점은 "학습"이 아니라 주어진 문맥 안에서만 해당 정보를 활용한다는 것입니다. 오픈북 시험처럼 책(검색된 문서)을 보면서 문제에 답하는 것과 같습니다.
3. **오답** - 사전학습은 Pre-training입니다.
4. **오답** - GPU 병렬 학습과는 다릅니다.
5. **오답** - 온라인에서도 가능합니다.

## 문제 6
### **RAG와 파인튜닝의 비교에서 올바른 것은?**

| 보기 | 설명 |
|------|------|
| 1. RAG는 모델 파라미터 내부에 지식을 저장한다 | |
| 2. 파인튜닝은 DB 내용만 변경하면 즉시 반영된다 | |
| **3. RAG는 외부 DB에서 검색 후 프롬프트에 추가하고, 파인튜닝은 모델 자체를 추가 학습한다** | |
| 4. RAG가 파인튜닝보다 항상 더 좋은 성능을 낸다 | |
| 5. 파인튜닝은 검색 과정이 필요하다 | |

**정답: 3**

**해설:**
1. **오답** - RAG는 외부 데이터베이스에 지식을 저장합니다. 모델 파라미터 내부 저장은 파인튜닝입니다.
2. **오답** - RAG가 DB 내용만 변경하면 즉시 반영됩니다. 파인튜닝은 재학습이 필요합니다.
3. **정답** - RAG는 외부 DB에서 검색 후 프롬프트에 추가하는 방식이고, 파인튜닝은 모델 자체를 추가 학습하여 파라미터를 변경하는 방식입니다.
4. **오답** - 상황에 따라 다릅니다.
5. **오답** - 파인튜닝은 검색 과정이 없습니다.

## 문제 7
### **RAG의 장점으로 **해당하지 않는** 것은?**

| 보기 | 설명 |
|------|------|
| 1. 실제 문서 기반으로 환각 감소 | |
| 2. DB 내용만 변경하면 즉시 반영 | |
| 3. 검색 + Context 처리로 응답 시간 증가 | |
| **4. 모델 파라미터가 변경되어 영구적으로 지식이 저장됨** | |
| 5. 최신 정보에 대응 가능 | |

**정답: 4**

**해설:**
1. **장점에 해당** - RAG는 실제 검색된 문서를 기반으로 답변하여 환각을 줄입니다.

3. **단점에 해당(장점은 아니지만 RAG의 특성)** - 검색과 Context 처리로 응답 시간이 늘어나는 것은 RAG의 단점입니다.
4. **정답(해당 안 함)** - RAG는 모델 파라미터를 변경하지 않습니다. 파라미터 변경은 파인튜닝의 특성입니다.

|------|------|

5. **오답** - RAG는 GPU 학습 비용이 필요 없습니다.
