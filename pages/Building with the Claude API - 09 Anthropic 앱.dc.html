Anthropic이 배포한 두 앱 — Claude Code(터미널 코딩 어시스턴트)와 Computer Use(컴퓨터 제어) — 를 에이전트 아키텍처의 실례로 다룬다. 두 앱은 에이전트 개념을 시연하는 실용적 예시다.

참고

Claude Code 심화는 별도 "Claude Code in Action" 자습 가이드를 참고하세요.

01·02

Anthropic 앱 & Claude Code 설치

Apps & setup

Claude Code = 코드 작업을 돕는 터미널 코딩 어시스턴트. 파일 검색/읽기/편집 + 고급 도구(웹 페칭·터미널) + MCP 클라이언트 지원.

  1. Node.js 설치(npm help로 확인)
  2. npm install로 Claude Code 설치
  3. 터미널에서 claude 실행 → Anthropic 계정 로그인
03

Claude Code 실전

Claude Code in action

단순 코드 생성기가 아니라 프로젝트의 협업 엔지니어처럼 동작(설계·작성·테스트·배포·오류 수정). init 커맨드로 코드베이스를 스캔해 CLAUDE.md 생성, #로 메모리 추가.

방법 1 — 3단계 워크플로우

① 관련 파일 식별·분석 → ② 해결책 계획만 요청(코드 X) → ③ 계획 구현.

방법 2 — TDD

① 컨텍스트 제공 → ② 테스트 제안 요청 → ③ 테스트 구현 → ④ 통과까지 코드 작성.

핵심 원칙

Claude Code = 노력 증폭기(effort multiplier). 상세한 지시 = 훨씬 나은 결과. 코드 생성기가 아니라 협업 엔지니어로 대하라.

04

MCP 서버로 기능 강화

Enhancements with MCP servers

Claude Code에는 MCP 클라이언트가 내장되어 코어 수정 없이 실시간으로 기능을 확장한다.

claude mcp add [server-name] [startup-command]
  • 예시: "Document Path to Markdown" 도구를 노출하는 서버 → Claude Code가 uv run main.py로 PDF/Word 읽기.
  • 흔한 사용처: 프로덕션 모니터링(Sentry), 프로젝트 관리(Jira), 커뮤니케이션(Slack).
  • 셋업: 서버 생성 → 이름·시작 명령으로 추가 → Claude Code 재시작.
05

Claude Code 병렬화

Parallelizing Claude Code

병렬화 = 여러 Claude 인스턴스를 동시에 돌려 다른 작업을 병렬 수행. 핵심 문제: 여러 인스턴스가 같은 파일을 동시 수정하면 충돌 발생.

해법 — Git Work Trees

서로 다른 Git 브랜치에 대응하는 별도 디렉터리에 프로젝트 복사본을 만들어 인스턴스마다 격리된 작업 공간 제공. 워크플로우: work tree 생성 → 작업 할당 → 격리 작업 → 커밋 → main 머지. 커스텀 커맨드(.claude/commands/·$ARGUMENTS)로 자동화.

이점: 개발자 한 명이 가상 엔지니어 팀을 지휘 — 머지 충돌·정리도 Claude가 처리.

06

자동 디버깅

Automated Debugging

AI로 프로덕션 오류를 수동 개입 없이 자동 탐지·분석·수정.

  1. GitHub Action이 매일 프로덕션 점검
  2. 최근 24시간 CloudWatch 로그 가져오기
  3. Claude가 오류 식별·중복 제거
  4. 각 오류 분석 후 수정 생성
  5. 제안 해결책으로 PR 생성

구성: GitHub Actions(스케줄) + AWS CLI(로그) + Claude Code(분석·수정) + CloudWatch(모니터링). 흔한 사례: 환경 간 설정 오류(로컬은 되지만 프로덕션 실패).

07·08

컴퓨터 사용 & 원리

Computer Use & how it works

Computer Use = Claude가 시각적 관찰 + 제어 동작으로 컴퓨터 인터페이스와 상호작용. 스크린샷 촬영·클릭·입력·탐색으로 QA 테스트·자동화를 수행.

사용자 지시
테스트 요구사항
→
도구 시스템 ↔ Claude
스크린샷 관찰 · action(클릭·타이핑) 요청
→
Docker 컨테이너
실제 키/마우스 실행
핵심 원리

Claude는 컴퓨터를 직접 조작하지 않는다. Computer Use = 도구 시스템 + 개발자가 제공하는 컴퓨팅 환경. 작은 스키마가 내부에서 mouse move·click·screenshot 등 action으로 확장된다. Anthropic이 참조 구현(Docker 컨테이너)을 제공한다.
