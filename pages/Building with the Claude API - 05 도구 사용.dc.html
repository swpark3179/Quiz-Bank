도구 사용 (Tool use)

도구 함수·스키마 · 메시지 블록 · 도구 결과 · 멀티턴/멀티툴 · 내장 도구

① 시작·개요② API 접근③ 프롬프트 평가④ 엔지니어링⑤ 도구 사용⑥ RAG·검색⑦ Claude 기능⑧ MCP⑨ Anthropic 앱⑩ 에이전트

도구 사용 = Claude가 학습 데이터 너머의 외부 정보에 접근하는 방법. Claude는 학습 데이터의 정보만 알고 현재/실시간 정보가 없으므로, 도구가 요청들 사이에서 외부 데이터 검색을 조율해 응답을 보강한다.

01

도구 사용 소개

Introducing tool use

도구 사용 흐름

  1. Claude에 초기 요청 + 외부 데이터 접근 지침 전송
  2. Claude가 외부 데이터 필요 여부를 판단하고 특정 정보 요청
  3. 서버가 코드를 실행해 외부 소스에서 데이터 가져옴
  4. 가져온 데이터와 함께 후속 요청
  5. Claude가 원래 프롬프트 + 외부 데이터로 최종 응답 생성

예(날씨): 사용자 질문 → Claude가 날씨 데이터 요청 → 서버가 날씨 API 호출 → Claude가 데이터 수신 → 정보 기반 응답.

03

도구 함수

Tool functions

도구 함수 = Claude가 추가 정보를 필요로 할 때 자동 실행되는 평범한 Python 함수.

def get_current_datetime(date_format="%Y%m%d %H:%M:%S"): if not date_format: raise ValueError("date format cannot be empty") return datetime.now().strftime(date_format)
모범 사례

서술적인 함수명·인자명을 쓰고, 입력을 검증해 의미 있는 에러 메시지와 함께 에러 발생 → 에러는 Claude에 보여서 수정된 파라미터로 재시도 가능하다.

04

도구 스키마

Tool schemas

도구 스키마 = 도구 함수와 매개변수를 모델에 설명하는 JSON 스키마 명세.

필드역할
name도구 식별자
description도구가 무엇을 하는지·언제 쓰는지·어떤 데이터를 반환하는지 3~4문장
input_schema인자의 타입·설명을 담은 실제 JSON 스키마
스키마 생성 트릭

도구 함수를 Claude.ai에 가져가 "이 함수의 유효한 도구 호출 JSON 스키마를 작성하라, 첨부 문서의 모범 사례를 따르라" + Anthropic tool use 문서 첨부 → 생성된 스키마 복사. 구현 시 anthropic.types의 ToolParam으로 감싸 타입 에러 방지.

05

메시지 블록 처리

Handling message blocks

도구를 포함한 요청은 tools 키워드 인자에 JSON 스키마를 담아 사용자 메시지와 함께 전송한다. 이제 메시지는 텍스트 블록만이 아니라 여러 블록을 포함한다.

  • Text block = 사용자용 설명 · Tool use block = 함수 이름 + 인자(도구 실행용).
  • 기록 관리: response.content(모든 블록)를 메시지 리스트에 추가(텍스트만 X). add_user/assistant_message가 여러 블록을 지원하도록 업데이트.
06

도구 결과 전송

Sending tool results

도구 결과 = 실행된 도구 함수의 결과를 후속 요청으로 Claude에 다시 전송.

도구 결과 블록 구조
필드의미
tool_use_id원본 tool use 블록 ID와 매칭(요청-결과 짝짓기)
content도구 출력을 문자열로 변환(보통 JSON)
is_error함수 실행 에러 여부 불리언(기본 false)
주의

후속 요청엔 완전한 기록 + 원본 도구 스키마를 포함해야 한다(도구를 다시 안 써도). ⚠️ 도구 결과 블록은 어시스턴트가 아니라 사용자 메시지에 들어간다.

07

도구를 이용한 멀티턴 대화

Multi-turn conversations with tools

도구 체이닝: 질문 → 첫 도구 요청 → 실행 → 결과 → 둘째 도구 요청 → 실행 → 결과 → 최종 답변. 예: "오늘부터 103일 후는 무슨 요일?" → get_current_datetime → add_duration_to_datetime → 답변.

핵심

질의가 몇 개의 도구를 필요로 할지 예측 불가하므로, 시스템이 임의 길이의 도구 호출 체인을 while 루프로 자동 처리해야 한다(run_conversation).

09

여러 도구 사용하기

Using multiple tools

초기 프레임워크 후 도구 추가 = 스키마 + 라우팅 + 구현의 단순 반복 패턴.

  1. run_conversation의 tools 리스트에 도구 스키마 추가
  2. run_tool에 새 도구 이름 처리 분기 추가
  3. 실제 도구 함수 구현

예: AddDurationToDateTime(날짜 계산) + SetReminder(목 구현). Claude는 한 대화에서 여러 도구를 순차 사용 가능(날짜 계산 후 그 결과로 리마인더 설정).

세밀한 도구 호출

Fine grained tool calling

도구 스트리밍은 input_json_delta 이벤트를 추가한다 — partial_json(청크) + snapshot(누적 합).

모드동작트레이드오프
기본청크 버퍼링 → 완전한 키-값 쌍까지 대기 → 스키마 검증 후 전송느리지만 검증된 JSON
Fine-grained
fine_grained: true		API측 JSON 검증 비활성화, 생성 즉시 청크 전송빠르지만 유효하지 않은 JSON 가능 · 클라이언트 에러 처리 필요

11

텍스트 편집 도구

The text edit tool

Text Editor Tool = 파일/텍스트 작업(읽기·쓰기·생성·치환·실행취소)을 위한 Claude 내장 도구. JSON 스키마만 내장돼 있고 실제 구현은 직접 코딩한다.

  1. 최소 스키마 스텁(name + 버전별 날짜 type)을 전송
  2. Claude가 내부에서 전체 스키마로 확장
  3. Claude가 도구 사용 요청 전송
  4. 커스텀 구현이 실제 파일 작업 실행(보기·문자열 치환·생성)
  5. 결과를 Claude에 반환
주의 · 용도

스키마 type 문자열은 Claude 모델 버전마다 다름(3.5 vs 3.7). 용도: AI 코드 에디터 기능 재현, 자동 코드 생성/리팩터링, 멀티파일 조작.

12

웹 검색 도구

The web search tool

Web Search Tool = 최신/전문 정보를 웹에서 검색하는 내장 도구. 커스텀 코드 불필요 — Claude가 검색을 자동 처리.

스키마 요구사항
  • type: "web_search_20250305"
  • name: "web_search"
  • max_uses: 검색 횟수 제한(기본 5)
  • allowed_domains: 도메인 제한(선택)
응답 구조
  • Text = Claude 설명
  • Tool use = 검색 쿼리
  • Web search result = 찾은 페이지(제목·URL)
  • Citation = 진술을 뒷받침하는 텍스트

예: 의료/운동 조언을 NIH.gov로 제한하면 일반 웹 대신 과학적 근거 기반 정보를 확보한다.
