Model Context Protocol (MCP)

MCP 서버/클라이언트 구축 · 도구 · 리소스 · 프롬프트

① 시작·개요② API 접근③ 프롬프트 평가④ 엔지니어링⑤ 도구 사용⑥ RAG·검색⑦ Claude 기능⑧ MCP⑨ Anthropic 앱⑩ 에이전트

MCP = Claude에 컨텍스트와 도구를 제공하는 통신 계층. 서비스마다 도구 스키마/함수를 직접 작성하던 부담을 전용 MCP 서버로 이전한다. 핵심은 도구·리소스·프롬프트 3대 프리미티브.

참고

이 섹션은 별도 강의 "Introduction to Model Context Protocol"과 동일한 커리큘럼입니다. 더 자세한 도식과 코드 예제는 별도의 MCP 자습 가이드 문서를 참고하세요.

Implementing a client

클라이언트 세션을 감싸 연결·리소스 정리를 관리하는 래퍼 클래스(connect/cleanup/async enter/async exit).

async def list_tools(): return (await self.session.list_tools()).tools async def call_tool(name, inp): return await self.session.call_tool(name, inp)

권장: 세션을 더 큰 클래스로 감싸 리소스 관리를 개선한다.

05

리소스 정의·접근

Defining & accessing resources
  • 리소스 = 클라이언트의 읽기 작업을 위해 서버가 데이터를 노출. 타입: 직접/정적(docs://documents), 템플릿형(docs://documents/{doc_id}).
  • 흐름: read resource 요청(URI) → 서버가 URI를 함수에 매칭 → 실행 → 결과 반환. @mcp.resource(URI + MIME 타입).
  • 접근: read_resource(uri) → result.contents[0] → mime_type이 json이면 json.loads, 아니면 resource.text.
리소스 vs 도구

리소스는 데이터를 선제적으로 제공(@ 멘션 시 문서 내용 가져오기), 도구는 반응적으로 행동 수행(Claude가 호출 결정 시).

06

프롬프트 정의·클라이언트

Defining prompts · in the client

프롬프트 = 서버가 노출하는 미리 작성·검증된 지침. 슬래시 커맨드(/format)로 접근. @prompt(name·description) + 인자 수용 + 메시지 리스트 반환.

async def list_prompts(): return (await self.session.list_prompts()).prompts async def get_prompt(name, args): return (await self.session.get_prompt(name, args)).messages

인자가 키워드 인자로 전달 → 프롬프트 텍스트에 보간 → 메시지 배열 반환(LLM에 직접 투입). 프롬프트 = 클라이언트가 인자로 호출하는 서버 정의 템플릿.
