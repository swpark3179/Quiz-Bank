01

프롬프트 엔지니어링 개요

Prompt engineering

언어 모델로부터 더 신뢰성 있고 고품질인 출력을 얻기 위해 프롬프트를 개선하는 것. 모듈 구조: 나쁜 프롬프트로 시작 → 기법 단계별 적용 → 각 기법 후 개선 평가.

  • 예시 목표: 키·몸무게·신체 목표·식이 제한 기반 운동선수용 하루 식단 생성.
  • 핵심 구성요소: prompt_input_spec(필요한 입력 정의), extra_criteria(모델 채점 추가 검증), output.html(점수 리포트). 동시성 지원(max_concurrent_tasks를 레이트 리밋에 맞춤).
02

명확하고 직접적으로

Being clear and direct

프롬프트 첫 줄에 행동 동사(action verb)로 정확한 작업을 간결·직접적으로 명시한다. 첫 줄은 AI 응답의 토대를 놓는 가장 중요한 부분.

구조

행동 동사 + 명확한 작업 설명 + 출력 명세. 예: "Generate a one day meal plan for an athlete that meets their dietary restrictions"

결과: 점수 2.32 → 3.92.

03

구체적으로

Being specific

구체성 = 출력을 특정 방향으로 유도하는 가이드라인/단계 추가.

Type A · 속성
원하는 출력의 품질·속성

길이·구조·포맷 나열 → 출력 특성 제어. 거의 모든 프롬프트에 권장.

Type B · 단계
모델이 따를 추론 단계

답에 도달하는 방식 제어. 넓은 관점이 필요한 복잡한 문제에.

두 기법은 전문 프롬프트에서 자주 결합된다. 효과: 점수 3.92 → 7.86으로 급등.

04

XML 태그로 구조화

Structure with XML tags

XML 태그 = 프롬프트 내 서로 다른 콘텐츠 섹션을 구분·조직해 AI 이해를 높이는 기법. 많은 콘텐츠를 보간할 때 정보 유형을 구별하게 돕는다.

<!-- 비구조 텍스트를 쏟아붓지 말고 서술적 태그로 감싼다 --> <my_code> ... 디버깅할 코드 ... </my_code> <docs> ... 참고 문서 ... </docs> <athlete_information> ... 외부 입력 ... </athlete_information>
명명 규칙

태그 이름은 구체적으로 — "data"보다 "sales_records"가 콘텐츠 성격에 대한 맥락을 준다. 콘텐츠 경계 혼동을 줄여 작은 블록에서도 품질이 향상된다.

05

예시 제공하기

Providing examples

One-shot / Multi-shot 프롬프팅 = 모델 행동을 유도하기 위해 예시를 제공(1개 / 여러 개). 예시를 XML 태그로 구조화해 샘플 입력과 이상적 출력을 담고 실제 프롬프트와 구분한다.

주요 활용

  • 코너 케이스 처리(빈정거림 감지, 엣지 시나리오)
  • 복잡한 출력 포맷팅(JSON 구조, 특정 포맷)
  • 기대 응답 품질·스타일 명확화
모범 사례

코너 케이스엔 맥락 추가("빈정거림에 특히 주의") · 이상적인 이유(reasoning)를 포함 · eval 최고 점수 예시를 템플릿으로 · 예시는 주요 지침 뒤에 배치. "무엇이 이 예시를 이상적으로 만드는가"의 설명을 곁들이면 효과가 강화된다.
